GUÍA TÉCNICA: GENERACIÓN DE DOCUMENTOS DOCX CON PRECISIÓN QUIRÚRGICA
========================================================================

Este documento describe el método exacto que utilizo para generar archivos .docx 
con formato institucional (UNIFRANZ / APA 7) sin errores de conversión.

1. LENGUAJE Y LIBRERÍAS CORE
----------------------------
Para evitar los errores de traducción de Pandoc o Typst hacia Word, yo uso:
- Lenguaje: Python 3.x
- Librería principal: 'python-docx' (permite manipular el XML de Office directamente).

2. REQUISITOS EN TU SISTEMA (Si quieres replicarlo)
--------------------------------------------------
Si deseas ejecutar estos scripts en tu propia computadora, necesitas:
- Instalar Python: https://www.python.org/
- Instalar la librería mediante terminal:
  pip install python-docx

3. EL MÉTODO DE "DIBUJO" (LÓGICA DE PROGRAMACIÓN)
-------------------------------------------------
En lugar de "convertir", el código sigue estos pasos lógicos:

A. Configuración del Lienzo (Page Setup):
   Se definen los objetos 'sections'. Aquí se establece el tamaño físico (Letter) 
   y los márgenes exactos en centímetros convertidos a Twips (la unidad de Word).

B. Definición de Estilos (CSS de Word):
   Se redefine el estilo 'Normal'. Se le asigna:
   - Font: Arial, 12pt.
   - Alignment: Justified.
   - Line Spacing: 1.5 (o 18pt).
   - Paragraph Format: First line indent de 1.25cm.

C. Manejo de Capítulos (Centrado Vertical):
   Para que un título aparezca en el centro de una hoja vacía:
   - Se inserta un 'Page Break'.
   - Se crea un párrafo con 'space_before' de aproximadamente 250pt a 300pt.
   - Se aplica tamaño 32pt y se fuerza el salto de página posterior.

D. Jerarquía de Títulos:
   - Heading 2: Bold.
   - Heading 3: Bold + Italic.
   Se programa para que no tengan colores azules (default de Word) sino negro.

E. Construcción de Tablas (Capítulo 4):
   Se generan mediante una matriz de datos. El script dibuja la tabla celda por celda.
   Para cumplir con APA 7:
   - Se eliminan las líneas verticales.
   - Se mantienen solo las líneas horizontales en el encabezado y el final.

4. FLUJO DE GENERACIÓN
----------------------
1. Entrada: Recibo tu prompt con instrucciones y archivos fuente (.typ, .pdf).
2. Procesamiento: Extraigo el texto y lo organizo en la estructura de capítulos.
3. Ejecución: Corro un script de Python que "dibuja" el documento en la memoria.
4. Salida: El comando doc.save('nombre.docx') empaqueta todo en el formato 
   OpenXML final que tú descargas.

5. ¿POR QUÉ NO USO PANDOC?
--------------------------
Pandoc es un conversor generalista. Este método es un "generador específico". 
Al programar el documento, me aseguro de que el diseño no se mueva al abrirlo 
en diferentes versiones de Microsoft Word.

¡Espero que esta información te sirva para tus estudios de Ingeniería!, "¡No te preocupes! Ese error 404 en **Arch Linux** ocurre porque tu lista de paquetes está desactualizada; estás intentando bajar una versión que ya no existe en los servidores porque ha sido reemplazada por una más nueva.

Para arreglarlo de un solo golpe, solo tienes que sincronizar tu sistema antes de instalar. Sigue estos pasos exactos:

### 1. Sincroniza e Instala `pip`
El flag `-S` solo instala, pero `-Sy` actualiza la base de datos de los servidores. Ejecuta esto:
```bash
sudo pacman -Syu python-pip
```
*(Si te pregunta si quieres actualizar todo el sistema, dile que **s** para evitar conflictos de versiones).*

### 2. Instala la librería para el documento
Una vez que `pip` termine de instalarse, ejecuta:
```bash
pip install python-docx --break-system-packages
```
*Nota: En Arch Linux moderno, es necesario añadir `--break-system-packages` para que te deje instalar librerías fuera de un entorno virtual (es seguro para este caso).*

### 3. Ejecuta tu generador
Ahora que ya tienes todo el "motor" listo, corre tu script:
```bash
python generar_informe_docx.py
```

---

### ¿Qué pasó con los errores 404?
En Arch Linux, los espejos (mirrors) se actualizan casi a diario. Cuando ejecutaste `pacman -S python-pip`, tu computadora buscaba la versión `26.1-1`, pero en los servidores quizás ya estaba la `26.1-2`. Al no coincidir, el servidor te devuelve el error **404 (No encontrado)**. El comando `-Syu` corrige esto actualizando las direcciones.



### Si `pacman` sigue fallando:
Si los servidores siguen dándote problemas, es posible que tus "mirrors" sean muy lentos o antiguos. Puedes refrescarlos rápido con:
```bash
sudo pacman -S reflector
sudo reflector --latest 5 --sort rate --save /etc/pacman.d/mirrorlist
```
Y luego intenta de nuevo el paso 1. **¡Prueba ahora y verás cómo el archivo .docx aparece por fin!**"
